# Style Guide Police

## Exercise Goal
Learn to use VS Code's diff and staging features by fixing formatting errors.

## Instructions
1. Clone this repository using VS Code's "Clone Git Repository" feature
2. Open `press_release.md` and fix the formatting errors
3. Use the Source Control sidebar to review your changes in the Diff View
4. Stage and commit your changes
5. Push to GitHub

## What to Look For
- Double spaces
- Missing or incorrect headers
- Misspelled product names
- Formatting inconsistencies
